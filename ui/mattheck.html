SKO (Topologie)

Entfernt unterspanntes Material bis zum Zielvolumen (Lager-/Kraft-Zonen bleiben). Moderate Voxelgröße = interaktiv.

SIMP-Pro (Methode 2 · scharfe Kanten)

Dreifeld-SIMP: Dichtefilter → Heaviside-Projektion (β-Continuation) + OC. Netzunabhängige, scharfe Member wie in topopt — Filterradius = Mindest-Stegbreite, β steuert die Kantenschärfe. FE-Löser identisch zur SKO, nur die Optimierungsschleife ist feiner.

Beulanalyse (Methode 3a · Stabilität)
Noch keine Analyse. Lastfall + Struktur (Vollteil, SKO oder SIMP-Pro) nötig.

Lineares Beulen: kleinster Buckling-Load-Factor λ (kritischer Lastfaktor) + erste Knickform. λ>1 = stabil bei aktueller Last (λ = Sicherheitsfaktor gegen Knicken); λ<1 = knickt vorher. Verifiziert gegen Euler (±6%). Amplitude-Regler zeigt die Knickform.

Beul-Optimierung (Methode 3b)
Maximiert den kleinsten Knick-Lastfaktor λ bei festem Volumen — verteilt Material gegen Knicken (druckdominierte Fälle). Multi-Mode-Eigenlöser + KS + OC, Sensitivität exakt gegen finite Differenzen verifiziert. Lastfall nötig. Läuft im Hintergrund; feine Gitter dauern.
CAO (Kerbrundung / Mattheck)

Vergrössert den Bauraum automatisch (obere Grenze je Achse), wenn CAO dort anwachsen möchte, statt am Rand abzubrechen. Nur bei Box-Bauraum wirksam (bei STL-Bauraum ohne Effekt).

Trägt Material an hochbelasteten Oberflächen auf, entfernt an unterbelasteten → gleichmäßige Oberflächenspannung. Auf Vollteil oder SKO-Ergebnis anwenden.

PicoGK-Kern (glattes STL)

Schickt das aktuelle Voxelfeld (nach SKO/CAO) an den lokalen PicoGK-Server → glattes STL, direkt angezeigt. „FEM auf Kern" nimmt das geglättete Ergebnis als neuen Bauraum und rechnet neu. Server starten: lattice-fraktal serve, Seite über http://localhost:5151/mattheck.html öffnen.

Lattice direkt (ohne Server)

Erzeugt das Gitter direkt aus dem SKO/CAO-Voxelfeld — kein Server nötig. BCC = isotrop (Zentrum→8 Ecken); „Lastpfad" richtet die Streben an den Hauptspannungen aus (braucht einen FEM-Lauf) und skaliert die Dicke mit der Spannung. STL = überlappende Zylinder (der Slicer vereinigt sie). Für massiv & wasserdicht den PicoGK-Kern nutzen.

→ Tessellation-Übergabe

Für Tessellation: Bauraum-STL = die SKO/CAO-Voxelform (dort als „＋ Bauraum-STL" laden → Gitter füllt die Form). Kraftfluss = getracte Hauptspannungs-Trajektorien als JSON (dort als Attraktor importieren). Glättung = Kurven weicher; Linie löschen = Modus an, dann unerwünschte Linie im 3D anklicken. Beide im selben Koordinatensystem. Kraftfluss braucht einen FEM-Lauf („Lösen").

→ Impliziter Kern (CAIO-Feld)

Implizites, wasserdichtes Gitter im PicoGK-Kern: Gyroid/Streben, Wandstärke aus von-Mises gegraded, Koordinaten am Hauptspannungs-Frame verzerrt (CAIO), an die Form geklippt. „Nur Form" = geglättete SKO/CAO-Voxelform (Marching-Cubes). Klippen auf: „Optimierte Form" füllt nur die SKO/CAO-Struktur; „Bauraum (Original-STL)" legt das Gitter über den gesamten Bauraum, an der groben FEM-Voxelform geklippt; „Bauraum scharf (STL-Mesh)" schickt die importierte Bauraum-STL an den Kern und schneidet das Gitter MESSERSCHARF an der Mesh-Oberfläche (feine Kern-Voxelgröße, unabhängig von der FEM-Auflösung). Der CAIO-Frame kommt aus dem FEM-Lauf. Braucht FEM-Lauf + laufenden Server. Anzeige/Speichern über die Kern-Buttons oben.

→ Booleans (Resultat ↔ Bauraum/STL)

Verrechnet das zuletzt gerenderte Kern-Resultat (A) mit dem Bauraum oder einer importierten STL (B): Vereinen, Abziehen, oder Split (innen/außen behalten). Wasserdicht im PicoGK-Kern. Braucht ein Kern-Resultat (z. B. „Feld-Lattice rendern") + laufenden Server. Ergebnis ersetzt die Anzeige und ist über „⬇ STL speichern" oben sicherbar.
